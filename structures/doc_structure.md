# 1. 规范文档结构

项目遵循严格的文档驱动开发流程，所有规范文档位于 `SpecMd` 目录下。

- `Specify.md`: **核心需求文档**。唯一的需求变更源与版本控制中心。
- `ProjectRules.md`: **通用准则**。全局适用的代码编写规则与底线。
- `Structure.md`: **项目架构**。包含项目目标、架构设计、技术栈及模块职责。
- `Specify_old.md` /`Specify_oldVersion.md`: 历史需求文档存档。
- `prompt_log.md`: 提示词迭代日志（可选）。
- `SpecMd/layers/`: **层级实现规范** 针对特定代码层（如数据层、模型层）的详细说明。
- `SpecMd/workflows/`: **流程与交互规范** 定义工作流及数据流转。
- `SpecMd/others/`: **其他规范** 依据项目变化而变化
- `SpecMd/agents/`: **智能体检查报告**
